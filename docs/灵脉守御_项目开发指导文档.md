# 《灵脉守御：修仙塔防》项目开发指导文档

版本：v0.1  
生成时间：2026-05-13

---

# 一、项目定位

## 游戏类型

- 修仙题材 Roguelike 塔防
- Survivor-like + Build 构筑 + 自动战斗
- Steam 平台单机买断制

---

# 二、核心设计目标

项目核心不是传统塔防，而是：

> “修仙世界中的阵法构筑与灵脉经营”

核心体验：

- 高频成长选择
- Build 联动爽感
- 五行阵法组合
- 自动战斗反馈
- 灵气流动视觉表现

---

# 三、核心玩法循环

## 白天阶段

- 探索地图
- 采集灵石
- 扩展灵脉
- 建造阵法
- 获取法宝

## 夜晚阶段

- 妖兽袭击
- 阵法自动战斗
- 法宝联动触发
- Boss / 天劫事件

## 战后阶段

- 三选一升级
- 灵根突破
- Build 强化
- 获取随机事件

---

# 四、核心系统设计

## 1. 阵法系统

五行阵法：

- 金阵
- 木阵
- 水阵
- 火阵
- 土阵

联动示例：

- 水 + 雷 = 感电
- 火 + 风 = 爆燃
- 木 + 土 = 灵植增殖

---

## 2. 法宝系统

法宝分类：

- 飞剑
- 灵珠
- 符箓
- 丹炉
- 御兽

法宝支持：

- 升级
- 词缀
- 超进化
- 元素标签

---

## 3. 灵根系统

玩家开局随机灵根：

- 金灵根
- 木灵根
- 水灵根
- 火灵根
- 土灵根
- 异灵根

灵根影响：

- 阵法方向
- 法宝成长
- Build 流派

---

## 4. Roguelike 系统

必须保证：

> 每 30~60 秒出现一次成长选择

成长内容：

- 新阵法
- 新法宝
- 词缀强化
- 灵脉升级
- 角色突破

---

## 推荐技术栈

# 七、开发阶段规划

## Phase 1：核心验证（1~2个月）

目标：

验证：

> “阵法 + 幸存者” 是否好玩

只做：

- 1 张地图
- 5 种阵法
- 20 个升级
- 10 分钟流程

---

## Phase 2：核心循环（2~4个月）

加入：

- Boss
- 灵根
- 事件
- 地图扩张
- 局外成长

---

## Phase 3：内容扩展

加入：

- 新流派
- 新法宝
- 新地图
- 新妖兽
- 新事件

---
